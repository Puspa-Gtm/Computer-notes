Introduction

Every computerized device needs specific commands to perform tasks. These commands, written in programming languages, instruct the computer to carry out desired actions. A set of instructions forms a program, and multiple programs are integrated to create software. In essence, software is a collection of interrelated programs that work together to perform specific functions.

Programs written by programmers are called source programs. Since computers do not directly understand these programs, they must be converted into machine language using language translators such as assemblers, compilers, and interpreters.

Qualities of Good Programs

  • Understandability: Good programs are easy to read and understand, with clear and logical structures that simplify comprehension and maintenance.
  • Accuracy: A reliable program must be error-free, performing its intended tasks without producing incorrect results or crashes.
  • Reliability: Programs should function correctly under various conditions, consistently delivering accurate outputs without unexpected failures.
  • User-Friendly Interface: A well-designed program offers an intuitive and straightforward interface that enhances the user experience and reduces learning curves.
  • Portability and Flexibility: The program should be adaptable to different systems with minimal modification, allowing it to run efficiently on various platforms.

Programming Language

Programming languages are essential for creating software applications and allowing users to communicate with computers. They consist of sets of keywords, variables, operators, loops, and other elements that are combined to perform specific tasks. Programmers use these languages to write instructions that guide computers on how to execute operations efficiently and accurately. Programming languages can be classified into low level and high level programming languages.

Low-Level Programming Languages

Low-level programming languages are closer to the hardware and are machine-dependent. These languages require a thorough understanding of computer architecture and are typically more difficult to write, debug, and maintain. They provide high execution speed and are essential for system programming and developing device drivers.

1. Machine-Level Language (1GL)

Machine-level language, also known as first-generation language (1GL), consists entirely of binary code (0s and 1s) directly understood by the computer's processor. It is highly efficient and fast, as there is no need for translation. However, it is also extremely challenging to program and debug due to its complex binary syntax. This language is machine-specific, meaning a program written for one system cannot run on another.

Advantages:

  • Directly executed by the processor, leading to high execution speed.
  • No need for language translators or compilers.
  • Efficient for hardware-specific tasks.

Disadvantages:

  • Difficult to write, understand, and debug due to binary code.
  • Machine dependency limits cross-platform compatibility.
  • Requires detailed knowledge of computer architecture.

2. Assembly Language (2GL)

Assembly language, or second-generation language (2GL), uses symbolic instructions known as mnemonics (like ADD, SUB, MOV) instead of binary code. It is a step above machine-level language but still hardware-specific. Assembly code is converted to machine language using an assembler. It is commonly used for embedded systems and low-level hardware programming.

Advantages:

  • Easier to write and debug compared to machine-level language.
  • Faster execution than high-level languages due to its proximity to hardware.
  • Useful for writing device drivers and real-time applications.

Disadvantages:

  • Machine dependent, making programs non-portable.
  • Complex and time-consuming to develop and maintain.
  • Requires knowledge of the hardware architecture.

High-Level Programming Languages

High-level programming languages are designed to be human-readable and closer to natural language. They are machine-independent and use compilers or interpreters to convert the code into machine language. These languages are commonly used for application development, data processing, and software engineering.

1. Procedural-Oriented Language (3GL)

Procedural-oriented languages, also known as third-generation languages (3GL), focus on defining the procedures or steps to achieve a task. They follow a top-down approach and are used to develop general-purpose applications. Examples include C, FORTRAN, and COBOL.

Advantages:

  • Easy to write, understand, and maintain due to structured programming.
  • Machine independent, allowing for cross-platform compatibility.
  • Efficient for complex calculations and data processing.

Disadvantages:

  • Less efficient than low-level languages for hardware control.
  • Requires a language translator, resulting in slower execution.

2. Object-Oriented Language (4GL)

Fourth-generation languages (4GL) emphasize data manipulation and ease of use, focusing on problem-solving rather than procedure definition. They support features like encapsulation, inheritance, and polymorphism. Examples include Python, Java, and Visual Basic.

Advantages:

  • More user-friendly with higher-level abstractions.
  • Supports rapid application development and prototyping.
  • Promotes reusability and modularity through object-oriented concepts.

Disadvantages:

  • Slower execution compared to low-level languages.
  • Requires more system resources.

3. Natural Language (5GL)

Fifth-generation languages (5GL) are designed to make programming closer to human language. They focus on problem-solving through logic and inference rather than explicit algorithmic steps. PROLOG is a well-known example.

Advantages:

  • Easier for non-programmers to write and understand.
  • Useful for artificial intelligence and machine learning applications.

Disadvantages:

  • Slow execution due to complex logic processing.
  • Limited in terms of performance and efficiency for low-level tasks.

Language Translators/Processors

Language translators are essential tools that convert code written in high-level or assembly languages (source code) into machine-level language (object code). Since machines cannot directly understand high-level code, translators bridge this gap by converting the code into binary format. There are three main types of language translators:

Assembler

An assembler translates assembly language programs into machine code. Since assembly language is close to machine code, the translation is relatively straightforward and fast. Assemblers convert the entire program at once and generate an object code file.

Compiler

A compiler converts high-level language programs into machine code all at once. It performs comprehensive syntax and semantic analysis and generates optimized machine code. Due to this, compiled programs generally execute faster, but the compilation process can take time. Examples of compiled languages include C, C++, and Java.

Interpreter

An interpreter translates high-level language code into machine code line by line during execution. This makes debugging easier since errors are detected one line at a time. However, interpreted programs tend to run slower than compiled ones. Popular interpreted languages include Python, Ruby, and PHP.

Difference between Compiler and Interpreter

S.N. Compiler Interpreter
1 It translates the whole program into object code at a time. It translates one line or a single statement of a program into object code at a time.

4 Compiler-based programs are difficult to code and debug. Interpreter-based programs are easy to code and debug.
5 It is usually a large program and requires larger memory space. It is usually a smaller program and requires less memory space.
6 New programming languages don’t use a compiler. New programming languages use an interpreter.

Syntax and Semantics

Syntax refers to the set of rules that define the structure of a programming language, while semantics deals with the meaning attached to each word or symbol within the code. For example, in QBASIC, the statement PRINT "This is programming"; P has PRINT as the semantic element and the entire line as the syntax.

Errors in Programming

An error in a program that prevents it from executing correctly is known as a bug. Debugging is the process of identifying and fixing these errors. Errors can arise from improper syntax, logic flaws, or runtime issues. The three main types of errors are:

  • Syntax Error: Occurs when code does not follow the correct syntax rules of the programming language. Detected by compilers or interpreters.
  • Logical Error: Occurs when the logic of the code is flawed, resulting in incorrect output. Not detected by language processors.
  • Runntime Error: Occurs while the program is running, often due to insufficient memory or device issues. Not detected by compilers or interpreters.

Common Terminologies in Programming

Data Types

Data types specify the type of data to be used in a program. Some common types are:

  • Numeric: Combination of positive, negative whole numbers, and zero.
  • Character: Combination of alphanumeric values and special symbols.

Variable and Constant

  • Variable: Holds numeric or alphanumeric values that change during program execution.
  • Constant: Holds numeric or alphanumeric values that do not change throughout program execution.

Operands and Operators

Operators are symbols that perform operations on operands. Some common types of operators are:

  • Arithmetic Operators: Perform mathematical calculations like addition (+), subtraction (-), multiplication (*), and division (/).
  • Relational Operators: Compare two values and return a Boolean result (e.g., ==, !=, >, <, >=, <=).
  • Logical Operators: Combine two or more conditions (e.g., AND, OR, NOT).
  • Assignment Operators: Assign values to variables (e.g., =, +=, -=, *=, /=).

Program Design Tools

There are several program design tools that are available. Some of the commonly used tools are algorithms, flowcharts, and pseudocode.

Algorithm

An algorithm is a step-by-step procedure or sequence of steps for solving a specific problem. Algorithms are written in simple English and are not tied to any programming language.

Rules for Writing an Algorithm:

  • Should execute output after a finite number of steps.
  • Should not depend on any programming language.
  • Should terminate after a finite number of steps.

Examples:

A. Algorithm to calculate simple interest:

  1. Start
  2. Read the values of P, T, R
  3. Calculate simple interest using: I = (P * T * R) / 100
  4. Display simple interest (I)
  5. Stop

B. Algorithm to find the greatest number between two numbers:

  1. Start
  2. Read any two numbers, a and b
  3. Check if (a > b)
  4. If yes, display 'The greatest number is a'
  5. If no, display 'The greatest number is b'
  6. Stop

Flowchart

A flowchart is a diagrammatic or pictorial representation of steps involved in an algorithm to solve a particular problem. It uses several geometrical shapes and symbols to represent different actions involved in problem-solving.

Types of Flowcharts:

  • Program Flowchart: Focuses on a specific problem and represents a single task or logic at a time.
  • System Flowchart: Shows the logical flow of a program and its collection, providing an overview of data flow and sequence of operations within a system.

What is C Programming Language?

C is a High-Level Programming Language, developed by Dennis Ritchie at AT&T’s Bell Laboratories in the USA in 1970. It is a very powerful programming language used for both system programming and application programming. Hence, it is often known as a Middle-Level Programming Language, as it supports features of both low-level and high-level languages.

Features of C Language:

  • Supports both high-level and low-level programming, making it a mid-level programming language.
  • Widely used for system programming and also popular for application programming.
  • Offers fast and efficient program development.
  • Includes powerful tools and keywords.
  • Has its own text editor and compiler.
  • Supports structured programming concepts.
  • Is easily portable across systems.
  • Provides a comprehensive library of functions.
  • Highly flexible for different programming needs.

    • Some Common Terminologies in C Programming

    • Identifiers: Names given to entities such as variables, arrays, functions, structures, unions, and labels.
    • Keywords: Reserved words that are predefined in the C language. There are 32 keywords as per ANSI, such as void, if, else, while, for, break, continue, etc.
    • Data Types: Specify the type of data to be used in a program. The primary types in C are:
      • int: Represents integer numbers without decimal points (2 bytes).
      • float: Represents real numbers with decimal points (4 bytes).
      • char: Represents a single character (1 byte).
      • void: Represents no value or null.
    • Constants: Fixed values that do not change during program execution.
    • Variables: Entities whose values can be changed during program execution.
    • Statements: Combinations of keywords, identifiers, operators, variables, and constants, ending with a semicolon or a curly bracket. They can be simple or compound statements.
    • Escape Sequences: Special character combinations like \n (new line), \t (tab), and \0 (null character) that have specific meanings.
    • Operators: Symbols used for performing mathematical and logical calculations. Operators can be unary (one operand), binary (two operands), or ternary (three operands). Examples include +, -, *, and /. They are categorized as:
      • Arithmetic Operators: Used for basic arithmetic operations such as addition (+), subtraction (-), multiplication (*), division (/), and modulus (%).
      • Relational Operators: Used to compare two values, returning true or false, e.g., ==, !=, >, <, >=, <=.
      • Equality Operators: Used to check equality (==) and inequality (!=) between values.
      • Logical Operators: Used to perform logical operations such as AND (&&), OR (||), and NOT (!).
      • Assignment Operators: Used to assign values to variables, e.g., =, +=, -=, *=, /=.
      • Increment/Decrement Operators: Used to increase (++) or decrease (--) the value of a variable by 1.
      • Conditional Operator (Ternary Operator): Used to evaluate a condition and return a value based on the result, e.g., condition ? value1 : value2.
      • Bitwise Operators: Used to manipulate individual bits of data, such as AND (&), OR (|), XOR (^), NOT (~), and bit shifts (<<, >>).
      • Special Operators: Include sizeof(), pointer operators (* and &), and member access operators (., ->).

    Control Structures in C:

    Control structures in C are constructs that define the flow of control in a program. They are essential for making decisions and performing repetitive tasks. There are two main types of control structures:

    1. Conditional Control Structures:

    Conditional control structures are used to make decisions based on conditions. They determine the execution path of a program by evaluating conditions and executing corresponding blocks of code. Common conditional control structures include:

    • if statement: The simplest conditional statement that executes a block of code if a specified condition is true. If the condition is false, the code block is skipped.
    • if-else statement: An extension of the if statement that provides an alternate block of code to execute when the condition is false.
    • switch statement: A multi-way branching statement that selects one of many code blocks to execute based on the value of an expression. It is often used when multiple conditions are checked for equality.

    2. Iterative Control Structures:

    Iterative control structures are used to execute a block of code repeatedly, either for a fixed number of times or while a given condition holds true. These loops are crucial when performing repetitive tasks, calculations, or processing data collections.

    • for loop: Typically used when the number of iterations is known beforehand. It consists of initialization, condition-checking, and increment/decrement, all in one line.
    • while loop: Continues to execute a block of code as long as the specified condition remains true. It is used when the number of iterations is not predetermined.
    • do-while loop: Similar to the while loop, but ensures that the code block is executed at least once, as the condition is checked after the execution.

    Arrays in C

    Arrays are collections of variables of the same data type stored in contiguous memory locations. They are used to store multiple values under a single name, allowing easy access and manipulation using an index. Arrays can be one-dimensional (linear) or multi-dimensional (like matrices). To declare an array in C, use the syntax:

    data_type array_name[size];

    For example:

    int numbers[5];

    Arrays are useful when the number of variables is large, as they allow efficient access and storage. They are commonly used for data processing, sorting, and storing sequences of elements.

    Strings in C:

    Strings in C are essentially arrays of characters, terminated by a null character '\0'. They are used to store sequences of characters and are manipulated using standard library functions like strlen(), strcpy(), strcmp(), and more. Strings can be declared as:

    char str[50];

    They are commonly used for storing text, displaying messages, and performing text processing. String handling is an essential part of programming when dealing with textual data.
